fix: remove orphaned code fragment causing esbuild parse error

The previous PowerShell regex replacement left a dangling code block
(the body of fetchYahooQuote) floating outside any function after
fetchRealPricesForSymbols. This caused esbuild to fail with
"Unexpected catch" at line 4229 because the orphaned try/catch block
was at the top level of the module scope.

Removed the 14 orphaned lines. server.ts now has exactly one
fetchYahooQuote, one fetchRealPricesForSymbols, and buildLiveMacroSnapshot
follows immediately after with no stray code between them.
