feat: EOD batch refresh — zero live HTTP during scans (CPU reduction)

Architecture change:
- Scan pipeline (buildUltraQuantDashboard + buildMultibaggerScan) now reads
  OHLCV and fundamentals from Supabase ONLY — no live Yahoo/Screener calls
- All live fetching moved to POST /api/admin/refresh-eod (call once at evening)
- /api/stock/enrich still does live Screener fetch (lazy, on row expand only)

New Supabase table: ohlcv_cache (symbol, candles JSONB, live_price, change_pct)
New functions in MarketDataAggregator:
  - loadOHLCVFromSupabase() — reads candles into realOHLCVCache
  - writeOHLCVToSupabase() — writes candles from refresh-eod

New endpoints:
  POST /api/admin/refresh-eod  — fetches top 150 symbols OHLCV + fundamentals
    body: { secret: "stockpulse-eod", symbols?: string[] }
  GET  /api/admin/refresh-eod/status — shows last refresh time + cached count

CPU impact: scan passes 2b/2c/3b removed (was 15+ live HTTP calls per scan)
Now scan = 2 Supabase reads (~50ms each) + zero Yahoo calls
