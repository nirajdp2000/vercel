fix: stocks showing SIM/no-price — fetch full OHLCV (range=2y) in Pass 2b

Root cause: Pass 2b was calling fetchYahooQuote (range=1d, ~5 candles)
which only populated perSymbolPriceCache but NOT realOHLCVCache.
fetchRealOHLCV requires >=60 candles, so realOHLCVMap stayed null
→ dataSource='synthetic', price='—'.

Fix: Pass 2b now calls fetchRealOHLCV (range=2y) for top 15 symbols
missing OHLCV. This gives full candles + live price in one call,
populating both realOHLCVCache and perSymbolPriceCache.
Pass 2c added as fallback: range=1d price-only for any still missing.
