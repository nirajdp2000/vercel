fix: extract live price from OHLCV meta for accurate real-time prices

Root cause: analyzeUltraQuantProfile was using enrichedData.yahoo.lastPrice
(from MarketDataAggregator enrichment cache) as the only real price source.
But enrichment is fire-and-forget and may not complete before first scan.

Fix: fetchRealOHLCV already calls Yahoo v8 chart API and gets meta.regularMarketPrice
in the same HTTP response as candles. Now we extract and store livePrice + changePct
from that meta field directly into realOHLCVCache.

Changes:
- realOHLCVCache now stores { candles, livePrice, changePct } — extracted from
  Yahoo v8 meta.regularMarketPrice in the same call that fetches candles
- perSymbolPriceCache populated from OHLCV fetch (avoids duplicate Yahoo calls)
- getLivePriceFromCache() helper returns livePrice + changePct from OHLCV cache
- analyzeUltraQuantProfile: new ohlcvLivePrice + ohlcvChangePct params
  Price priority: Yahoo enrichment > OHLCV meta livePrice > OHLCV last candle > null
- lstmPredictedPrice: scaled to realBase (live price) when it differs from endPrice
  (EOD close) by >0.5% — ensures LSTM target is relative to current live price
- Superbrain: receives best available real price (enrichment > ohlcvLivePrice > EOD)
- buildMultibaggerScan: same livePrice priority fix for currentPrice + pChange
- fetchYahooFundamentals: range=1y → range=5d (faster, smaller payload)
  52W high/low now from meta.fiftyTwoWeekHigh/Low (reliable in v8 meta)
- perSymbolPriceCache moved before fetchRealOHLCV to fix declaration order
- Added /api/debug/enrichment endpoint: shows cache state for any symbols
  (enrichment hit, OHLCV hit, livePrice, changePct, expiresIn)
